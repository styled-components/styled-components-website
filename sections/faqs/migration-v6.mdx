## What do I need to do to migrate to v6?

> v6.3+ adds native React Server Components (RSC) support: styled components work in RSC without `'use client'` directives or registry setup. v6.4 expands RSC coverage with `createTheme()`, `stylisPluginRSC`, and first-class `StyleSheetManager` behavior. See [Notable changes in v6.4](#notable-changes-in-v64) below and the [RSC section](/docs/advanced#react-server-components) for details.

First, let's start by updating the necessary packages to their latest versions.

If you're using NPM:

```sh
npm install styled-components@^6 stylis@^4
npm uninstall @types/styled-components
```

If you're using Yarn:

```sh
yarn add styled-components@^6 stylis@^4
yarn remove @types/styled-components
```

As styled-components now provides its own types, there's no longer a need for community ones.

### TypeScript

Good news for TypeScript enthusiasts – styled-components is now natively written in TypeScript! Even if you haven't used TypeScript before, it's recommended for improving the reliability of your project as it can alert you when you're using an unknown prop or if a prop has a different value than expected.

However, if you don't use TypeScript in your project, don't worry! IDEs like VS Code will still pick up the types and provide hints as you write your code.

### `shouldForwardProp` is no longer provided by default

If you haven't migrated your styling to use [transient props (`$prefix`)](/docs/api#transient-props), you might notice React warnings about styling props getting through to the DOM in v6. To restore the v5 behavior, use `StyleSheetManager`:

```tsx
import isPropValid from '@emotion/is-prop-valid';
import { StyleSheetManager } from 'styled-components';

function MyApp() {
    return (
        <StyleSheetManager shouldForwardProp={shouldForwardProp}>
            {/* other providers or your application's JSX */}
        </StyleSheetManager>
    )
}

// This implements the default behavior from styled-components v5
function shouldForwardProp(propName, target) {
    if (typeof target === "string") {
        // For HTML elements, forward the prop if it is a valid HTML attribute
        return isPropValid(propName);
    }
    // For other elements, forward all props
    return true;
}
```

### Vendor prefixes are omitted by default

As the web and browsers have matured significantly by 2023, vendor prefixing is often unnecessary. Therefore, for the v6 release, we've decided to omit automatic prefixing by default to reduce the amount of CSS delivered to a page. If you prefer the v5 behavior, you can restore it via `StyleSheetManager`:

```tsx
import { StyleSheetManager } from 'styled-components';

function MyApp() {
    return (
        <StyleSheetManager enableVendorPrefixes>
            {/* other providers or your application's JSX */}
        </StyleSheetManager>
    )
}
```

To accommodate this change, the original `disableVendorPrefixes` prop was inverted to `enableVendorPrefixes`; if you have `disableVendorPrefixes` set, you can now remove it as it's the new default.

### Update stylis plugins

styled-components v6 uses the newer stylis v4; if you are providing `stylisPlugins` to `StyleSheetManager`, ensure the plugins are up-to-date. For instance, [`stylis-plugin-rtl`](https://www.npmjs.com/package/stylis-plugin-rtl) released a new version to support the updated stylis.

### Nested syntax handling

With the stylis v4 upgrade [came a change to how nested selectors are handled](https://github.com/thysultan/stylis/issues/323) which now correctly mirrors browser behavior. Specifically, pseudoselectors (e.g. `:before`) that do not start with `&` will not have the ampersand implicitly added anymore.

v5 behavior

```tsx
styled.div`
  :hover { color: red; }
`
// .[classname]:hover { color: red; }

styled.div`
  &:hover { color: red; }
`
// .[classname]:hover { color: red; }
```

v6 behavior

```tsx
styled.div`
  :hover { color: red; }
`
// .[classname] :hover { color: red; } (equivalent to .[classname] *:hover)

styled.div`
  &:hover { color: red; }
`
// .[classname]:hover { color: red; }
```

### Transient `$as` and `$forwardedAs` props have been dropped

To reduce confusion around application order, we've dropped the transient `$as` and `$forwardedAs` props. Please use the regular `as` and `forwardedAs` props instead.

### Dropped legacy `withComponent()` API

This change has been a long time coming. The `withComponent` API is no longer supported, so please use the `as` prop instead. You can specify `as` at definition time via `attrs` or at runtime:

```tsx
import styled from 'styled-components';

const Button = styled.button`
    background: blue;
    color: white;
`;

const ButtonLink = styled(Button).attrs({ as: 'a' })``;

// These are equivalent, but `ButtonLink` allows for attaching further styling if desired.
<Button as="a" href="https://styled-components.com">
<ButtonLink href="https://styled-components.com">
```

### Minimum Node support raised to v16+

In line with the [maintenance schedule](https://github.com/nodejs/release#release-schedule) for Node, we now support v16 as the oldest runtime that's still receiving security patches.

### Notable changes in later v6.x releases

#### Babel macro removed (v6.1.0)

The `babel-plugin-macros` integration was removed due to low usage. If you were importing from `styled-components/macro`, switch to the standard import and use the Babel or SWC plugin directly.

#### Streaming SSR support (v6.2.0)

Experimental `renderToPipeableStream` support was added for React 18+ streaming SSR. See the [streaming rendering](/docs/advanced#streaming-rendering) section for details.

#### React Server Components (v6.3.0)

Styled components now work in RSC environments without `'use client'` directives. `ThemeProvider` is a pass-through in RSC (no React context). `StyleSheetManager` works in RSC (v6.4+); `stylisPlugins` and `shouldForwardProp` are applied. Use [`createTheme()`](/docs/api#createtheme) or CSS custom properties for theming instead of `ThemeProvider` in server components. See the [RSC section](/docs/advanced#react-server-components) for best practices.

#### New HTML & SVG element helpers (v6.3.0)

Added `styled.search`, `styled.slot`, `styled.template`, and SVG filter primitive helpers (`styled.clipPath`, `styled.linearGradient`, `styled.feGaussianBlur`, etc.).

#### CSS custom properties in TypeScript (v6.3.0)

`--var` style props now work without TypeScript errors in `.attrs()` and JSX `style` props.

### Notable changes in v6.4

#### IE11 build target removed

Internet Explorer 11 has been unsupported on v6 [since August 2021](https://github.com/styled-components/styled-components/issues/3333), when the maintainers publicly committed to dropping it for the v6 line. v6.0–6.3 still shipped an ES5 build as an incidental side-effect of the TypeScript target; v6.4 aligns the build to ES2015 to match the stated policy and [recover the bytes](https://github.com/styled-components/styled-components/issues/4292) that were being spent on an unsupported target. React 18 (the peer floor on v6) dropped IE11 itself, and Microsoft retired IE11 in June 2022.

If you still need to ship to IE11 in 2026, stay on v5. See [Browser Support](/docs/faqs#which-browsers-are-supported) for the full matrix.

#### `.attrs()` provides optional props

Props that `.attrs()` supplies a value for are now automatically optional on the component's type. See [Attaching additional props](/docs/basics#attaching-additional-props).

#### Immutable attrs callback props

The props object passed to an `.attrs(props => ...)` callback is now an immutable snapshot. If you were mutating it to pass values to later attrs layers, switch to returning those values from the callback instead:

```tsx
// Before (silently worked, now a no-op)
styled.button.attrs(props => {
  props.type = 'button';
  return {};
})``;

// After
styled.button.attrs(props => ({
  type: 'button',
}))``;
```

#### `createTheme()` for RSC-compatible theming

A new `createTheme(defaultTheme, options?)` helper produces a theme object backed by CSS custom properties. It works in both client components and React Server Components, unlike `ThemeProvider`, which is a pass-through in RSC. See the [`createTheme` API reference](/docs/api#createtheme).

#### `StyleSheetManager` works in RSC

Previously a no-op outside client components, `StyleSheetManager` now applies its `stylisPlugins`, `shouldForwardProp`, and `nonce` props in RSC environments.

#### `stylisPluginRSC` for child-index selectors

A new opt-in stylis plugin that rewrites `:first-child`, `:last-child`, and `:nth-child()` selectors so they ignore inline `<style>` tags emitted by RSC. See the [child-index selector caveat](/docs/advanced#child-index-selector-caveat) for when you need it.

#### CSP nonce auto-detection

styled-components now reads a CSP nonce from `<StyleSheetManager nonce>`, `new ServerStyleSheet({ nonce })`, `<meta property="csp-nonce">`, `<meta name="sc-nonce">`, or `__webpack_nonce__`. See [Security](/docs/advanced#content-security-policy-csp-nonces).

#### Faster re-renders

Styled components skip style work entirely on re-render when their props and theme are unchanged, so placing them under frequently-rerendering parents no longer incurs a per-render cost.

#### React Native fixes

`react-native` is now an optional peer dependency. React Native apps using Metro/Expo with `nanoid` no longer crash on startup (see [#5705](https://github.com/styled-components/styled-components/issues/5705)).
