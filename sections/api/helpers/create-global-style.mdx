### `createGlobalStyle` | v4 | web-only

A helper function to generate a special `StyledComponent` that handles global styles. Normally, styled components are automatically scoped to a local CSS class and therefore isolated from other components. In the case of `createGlobalStyle`, this limitation is removed and things like CSS resets or base stylesheets can be applied.

> React Server Components (v6.3.0+): `createGlobalStyle` works in RSC environments. Static global styles are injected once via inline `<style>` tags. In React 19, global styles render without the `precedence` attribute to avoid premature deduplication. `createGlobalStyle` is also StrictMode-safe as of v6.3.7.

Returns a `StyledComponent` that does not accept children. Place it at the top of your React tree and the global styles will be injected when the component is "rendered".

```tsx
import { createGlobalStyle } from 'styled-components'

const GlobalStyle = createGlobalStyle<{ $whiteColor?: boolean; }>`
  body {
    color: ${props => (props.$whiteColor ? 'white' : 'black')};
  }
`

// later in your app

<React.Fragment>
  <GlobalStyle $whiteColor />
  <Navigation /> {/* example of other top-level stuff */}
</React.Fragment>
```

Since the `GlobalStyle` component is a `StyledComponent`, that means it also has access to theming from the [`<ThemeProvider>` component](/docs/api#themeprovider) if provided.

```tsx
import { createGlobalStyle, ThemeProvider } from 'styled-components'

const GlobalStyle = createGlobalStyle<{ $whiteColor?: boolean; }>`
  body {
    color: ${props => (props.$whiteColor ? 'white' : 'black')};
    font-family: ${props => props.theme.fontFamily};
  }
`

// later in your app

<ThemeProvider theme={{ fontFamily: 'Helvetica Neue' }}>
  <React.Fragment>
    <Navigation /> {/* example of other top-level stuff */}
    <GlobalStyle $whiteColor />
  </React.Fragment>
</ThemeProvider>
```

> Avoid `@import` in `createGlobalStyle`. CSS `@import` rules placed inside `createGlobalStyle` may appear to work in development but will silently fail in production. This happens because styled-components uses CSSOM APIs (`insertRule`) in production, and browsers require `@import` rules to appear before all other rules in a stylesheet — an ordering that CSSOM injection cannot guarantee. Place `@import` statements in your HTML `<head>` instead:
>
> ```html
> <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Roboto" />
> ```
