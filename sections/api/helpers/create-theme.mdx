### `createTheme` | v6.4 | web-only

A helper function that converts a plain theme object into one where every leaf value is a CSS custom property reference (`var(--prefix-path, fallback)`). This enables theming that works in both client components and React Server Components without relying on React context.

> `createTheme` is the recommended approach for theming in RSC environments where [`ThemeProvider`](/docs/api#themeprovider) has no effect (no React context available).

      1. <Code>defaultTheme</Code>
    </Column>
    <Column>A plain object with your theme tokens. Nested objects are supported; paths become hyphenated variable names.</Column>
  </Row>
  <Row>
    <Column>
      2. <Code>options?</Code>
    </Column>
    <Column>
      Optional configuration: <Code>prefix</Code> (default <Code>"sc"</Code>) and <Code>selector</Code> (default <Code>":root"</Code>, use <Code>":host"</Code> for Shadow DOM).
    </Column>
  </Row>
</Table>

Returns a theme object with the same shape where every leaf is a `var()` reference. The returned object also has these properties:

<Table head={['Property', 'Description']}>
  <Row>
    <Column><Code>raw</Code></Column>
    <Column>The original plain theme object.</Column>
  </Row>
  <Row>
    <Column><Code>vars</Code></Column>
    <Column>Same shape as the theme, but every leaf is the bare CSS custom property name (e.g. <Code>--sc-colors-primary</Code>). Use in <Code>createGlobalStyle</Code> for dark mode overrides without hand-writing variable names.</Column>
  </Row>
  <Row>
    <Column><Code>resolve(el?)</Code></Column>
    <Column>Client-only. Reads computed CSS custom property values from the DOM and returns a plain object with resolved values. Optionally pass a DOM element to resolve against (defaults to <Code>document.documentElement</Code>).</Column>
  </Row>
  <Row>
    <Column><Code>GlobalStyle</Code></Column>
    <Column>A <Code>createGlobalStyle</Code> component that emits the CSS custom property declarations on the configured selector.</Column>
  </Row>
</Table>

```tsx
import styled, { createTheme } from 'styled-components';

const theme = createTheme({
  colors: {
    primary: '#0070f3',
    accent: '#7928ca',
    text: '#1a1a1a',
  },
  spacing: {
    sm: '8px',
    md: '16px',
    lg: '32px',
  },
});

// Each leaf becomes a CSS var reference:
// theme.colors.primary → "var(--sc-colors-primary, #0070f3)"
// theme.spacing.md     → "var(--sc-spacing-md, 16px)"

const Button = styled.button`
  background: ${theme.colors.primary};
  color: white;
  padding: ${theme.spacing.sm} ${theme.spacing.md};
`;
```

Pass the theme object to [`ThemeProvider`](/docs/api#themeprovider) for stable class name hashes across themes. This prevents hydration mismatches when switching between light and dark mode:

```tsx
<ThemeProvider theme={theme}>
  <App />
</ThemeProvider>
```

#### Composition rules

Theme tokens are CSS variables on web and sentinel strings in React Native. They aren't the original raw values, so they only resolve when used in CSS-value positions inside a styled-components template. The single rule: interpolate tokens directly; don't combine them with JavaScript arithmetic.

```tsx
const theme = createTheme({
  space: { md: '16px', lg: '32px' },
  colors: { primary: '#0070f3' },
});

const Card = styled.div`
  // works: direct interpolation
  padding: ${theme.space.md};
  color: ${theme.colors.primary};

  // works: multi-value
  margin: ${theme.space.md} ${theme.space.lg};

  // works: function position resolves at render
  background: ${({ $active }) => ($active ? theme.colors.primary : 'transparent')};

  // works: CSS calc() composes tokens with raw numbers
  top: calc(${insets.top}px + ${theme.space.lg});
`;
```

Why JS arithmetic breaks: on web `theme.space.md` is the literal string `"var(--sc-space-md, 16px)"`, so `4 + theme.space.md` produces `"4var(--sc-space-md, 16px)"`. The browser sees malformed CSS and silently drops the declaration. On native the token is a sentinel string with the same coercion failure, caught by a dev-time warning.

```tsx
// breaks on both platforms
top: ${insets.top + theme.space.md};
top: ${({ theme }) => 4 + theme.space.md}px;
```

The two correct fixes:

```tsx
// 1. Compose in CSS via calc()
top: calc(${insets.top}px + ${theme.space.md});

// 2. Reach for the raw object when you genuinely need a JS number
const offset = insets.top + theme.raw.space.md;  // 16 (number)
```

`theme.raw` exposes the original input object unchanged. Use it sparingly; theme overrides applied via `<ThemeProvider>` won't be reflected there because the cascade lives on the CSS side.

#### Dark mode

Use `vars` to reference CSS custom property names in your dark mode overrides. No hand-writing variable names:

```tsx
import { css, createGlobalStyle, createTheme } from 'styled-components';

const theme = createTheme({
  colors: { bg: '#ffffff', text: '#1a1a1a' },
});
const { vars } = theme;

const darkVars = css`
  ${vars.colors.bg}: #111827;
  ${vars.colors.text}: #f9fafb;
`;

const DarkOverrides = createGlobalStyle`
  @media (prefers-color-scheme: dark) {
    :root:not(.light) { ${darkVars} }
  }
  .dark { ${darkVars} }
`;

// Mount both: GlobalStyle emits :root vars, DarkOverrides adds dark mode
<theme.GlobalStyle />
<DarkOverrides />
```

This approach is CSS-first: the `@media (prefers-color-scheme: dark)` rule applies before any JavaScript runs, so there's no flash when following the system preference. See [Advanced Theming](/docs/advanced#dark-mode) for persisting a manual toggle without flash.

#### Custom prefix and selector

```tsx
const theme = createTheme(
  { colors: { primary: 'blue' } },
  { prefix: 'myapp', selector: ':host' }
);
// theme.colors.primary → "var(--myapp-colors-primary, blue)"
```
